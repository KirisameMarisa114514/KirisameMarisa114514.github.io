从 Hexo 到 GitHub Pages：Butterfly 二次元博客搭建与动效实践

Created2026-07-19|Updated2026-07-19|开发HexoFrontend
|Word Count:995|Reading Time:3mins|Post Views:

这次把一个已经初始化好的 Hexo 站点改造成了更符合个人风格的二次元技术博客。目标不是反复更换主题，而是在保留文章和部署流程的前提下，把主题配置、静态资源和自定义效果整理成一套可维护的结构。

自定义样式 原生 CSS

用站点级配置覆盖主题

Butterfly 支持站点根目录下的 _config.butterfly.yml。头像、导航、背景、文章卡片和搜索等个性化配置放在这里，比直接修改主题模板更容易升级。

这次实践的收获

这次改造最重要的并不是增加了多少动画，而是把职责划分清楚：

  • 文章只放在 source/_posts；
  • 图片和脚本只放在 source；
  • 主题个性化优先使用 _config.butterfly.yml；
  • 修改前先备份，修改后用构建和浏览器共同验证；
  • 动画要有粒子上限、移动端降级和后台暂停机制；
  • public 可以随时重新生成，不能作为唯一素材目录。

有了这套结构，后续无论增加 Live2D、音乐播放器还是新的文章栏目，都不需要破坏主题核心文件。
